# Project memory — Territory Developer

Index of architectural decisions and durable context that doesn't fit cleanly in
specs, commit messages, or BACKLOG rows. One line per entry. Promote an entry to
its own file under `.claude/memory/{slug}.md` only when it grows past ~10 lines.

Format: `- [Title](path-or-anchor) — one-line hook`

## Architecture decisions

- M6 lifecycle collapse rev 3 (2026-04-19) — single-level Stage/Task flow; 4 canonical pair seams (`stage-file-plan/apply`, `project-new/apply`, `plan-review/fix-apply`, `opus-code-review/code-fix-apply`, `stage-closeout-plan/apply`) + 2 Stage-scoped bulk Opus non-pair passes (`/author` plan-author 1×N, `/audit` opus-audit 1×N). Retired: `/kickoff` + `spec-kickoff` + `project-spec-kickoff` (folded into `plan-author`); `project-stage-close` + `project-spec-close` (folded into Stage-scoped `/closeout` pair). Per-Task seams = `/implement`, `/verify-loop`, `/code-review`. Canonical hard rule: `ia/rules/agent-lifecycle.md`; full matrix: `docs/agent-lifecycle.md`.
- `/ship-stage` default for multi-task stages (2026-04-18; updated 2026-04-20) — after `/stage-file` files ≥2 tasks in Stage X.Y, run `/plan-review` (+ `/plan-fix-apply` if needed) → `/author` → `/ship-stage {MASTER_PLAN_PATH} Stage {X.Y}`. **`/ship-stage` does not invoke `/author`.** Ship-stage Pass 1 = per-task implement + `unity:compile-check` + atomic commit; Pass 2 = batched `verify-loop` + stage code-review + audit + closeout. Escape hatch: `--per-task-verify` for large/wide stages. Canonical: `ia/skills/ship-stage/SKILL.md`, `ia/rules/agent-lifecycle.md`.
- `validate:all` vs sibling master plans (2026-04-20) — `validate:all` runs `validate:master-plan-status` across **all** orchestrators under `ia/projects/*master-plan*.md`. A red CI / verify-loop while shipping plan A may be drift in plan B; triage with `npm run validate:master-plan-status` and fix the failing row’s plan.
- Cursor `/ship-stage` + commits (2026-04-20) — subagent output may include a “suggested commit” block without performing `git commit`; confirm `git status` and commit (or squash) before opening a PR.
- Umbrella rollout tracker pattern (2026-04-17) — multi-bucket master plans (e.g. `ia/projects/full-game-mvp-master-plan.md`) pair with a sibling `ia/projects/{umbrella-slug}-rollout-tracker.md` that tracks each child through a 7-column lifecycle (a) enumerate → (b) explore → (c) plan → (d) stage-present → (e) stage-decomposed → (f) task-filed → (g) align. Cell glyphs: `✓` done, `◐` partial, `—` not started, `❓` ambiguous, `⚠️` disagreement. Target = column (f) ≥1 task filed per row; (g) is alignment gate (glossary + spec anchor + MCP resolves). Driven by `/release-rollout {UMBRELLA_SPEC} {ROW_SLUG}` which dispatches per-cell into existing lifecycle subagents — never reimplements decomposition / filing. Seeder: `release-rollout-enumerate` (one-shot).
- Per-skill Changelog convention (2026-04-17) — `ia/skills/{name}/SKILL.md` carries `## Changelog` at tail for fix audits (`### YYYY-MM-DD — {summary}` → bullets w/ symptom / fix / location / status-applied {commit|pending}). Tracker aggregator (`## Skill Iteration Log`) cross-references per-skill entries via anchor link. Dual-write enforced by `release-rollout-skill-bug-log` helper skill.
- Parallel-work rule for sibling orchestrators (2026-04-17) — NEVER run `/stage-file` or `/closeout` against two sibling master-plans concurrently on same branch. `release-rollout` umbrella skill enforces via Tier A→B/B'→C→D→E ordering; alt row surfaced when primary pick violates.

## CLI / tooling tips

- macOS `sed` does not support `\b` word boundaries; use `perl -pi -e 's/\bFoo\b/Bar/g'` for symbol renames from CLI agents.
- Bash `${VAR:+value}` expands on **non-empty** — `"0"` is non-empty, so boolean-like `CHECK_MODE=0` still triggers expansion. Use value-equality guard `[ "$VAR" = "1" ] && EXTRA_ARGS+=(--flag)` + array expansion for boolean flags (BUG-58 fix on `tools/scripts/materialize-backlog.sh`).
- `git mv` on a Unity `.cs` file leaves the adjacent `.meta` at the original name; `git mv` the `.meta` separately to preserve the GUID (prefab / scene references survive).
- Bridge `get_compilation_status` is a reliable compile gate when the Unity Editor holds the project lock and batchmode is blocked.
- macOS `flock` (Homebrew util-linux) installs to `/opt/homebrew/Cellar/util-linux/*/bin/` — not on npm test shell PATH by default. Tools spawning `reserve-id.sh` (or any flock-using script) from Node must patch `PATH` via a `buildScriptEnv()` / `scriptEnv()` helper mirroring `tools/scripts/reserve-id-concurrent.sh`, else tests + MCP handlers fail ENOENT on flock. Pattern applied in `tools/mcp-ia-server/src/tools/reserve-backlog-ids.ts`.
- Before filing a follow-up TECH from a project-spec audit, grep `BACKLOG.md` for an existing matching scope — duplicates waste id space and split tracking (TECH-04 already covered the Stage 1.2 invariant #5 cleanup scope).
- `InterstateManager.GenerateAndPlaceInterstate()` is the canonical single-call entry for scripted interstate builds; internally runs `RoadManager.PlaceInterstateFromPath` → `TryPrepareRoadPlacementPlan` → `PathTerraformPlan.Apply` → `InvalidateRoadCache()` → `NeighborCityBindingRecorder.RecordExits`. Satisfies invariants #2 + #10 — avoid direct `ComputePathPlan`.
- `MapGenerationSeed.SetSessionMasterSeed(int)` pins deterministic new-game seeds for testmode (consumed by `AgentTestModeBatchRunner -testSeed N`).
- Before authoring a spec that introduces a new static-helper class, grep the target namespace for the proposed name — nested patch-data structs (e.g. `BlipEnvelope` inside `BlipPatchTypes.cs`) collide w/ top-level type declarations and only surface at compile time (CS0101). Suffix helpers w/ `-Stepper` / `-Builder` / `-Service` when the bare noun already exists.
- Edit tool replacing a `for` loop header: include full body in `old_string`. Header-only replace orphans body block.
- `validate:frontmatter` = advisory mode (exits 0 on warnings). Gate on stdout inspection, not exit code, when enforcing "zero issues". Hard gate for stale field removal = `grep -r "{field}" ia/projects/` (cheaper + deterministic).
- `AutoZoningManager.ZoneSegmentStrip`: loop upper bound + segment removal condition (`zonedUpToIndex >= L - N`) are two sites that must stay consistent — update both when extending endpoint coverage.
- `SelectZoneTypeForSegment(CompletedSegment)` → extract `SelectZoneTypeForRing(UrbanRing)` to decouple ring-based demand logic from segment struct; reuse from frontier rescan paths.
- Stage-compress to single issue when every task is ≤1 file or docs-only and none kicked off — n→1 collapse cuts step overhead w/ zero sunk cost (first applied to web-platform Stage 1.1, 2026-04-14).
- Root npm script composition: prefer `npm --prefix {dir} run …` over `cd {dir} && …` — reliable exit-code propagation, no subshell state quirks.
- Vercel project link + first deploy are dashboard-only (no CLI auth in agent env) — pre-note as `[HUMAN ACTION]` in Phase plan before implementation, not mid-phase.
- `npx create-next-app@latest` auto-generates `AGENTS.md` + `CLAUDE.md` under target dir — review/overwrite post-scaffold to match repo conventions.
- `validate:all` runs `validate:backlog-yaml` before any TS compile step — `.mjs` validator importing TS dist would be stale; rename to `.ts` + run via `npx tsx` for zero-config shared-core reuse (TECH-323, 2026-04-17).
- Near-duplicated yaml parsers (`parseYamlScalars` in `tools/validate-backlog-yaml.ts` + `tools/mcp-ia-server/src/parser/backlog-yaml-loader.ts`) are a drift risk — consolidate to one canonical parser when extracting shared lint cores (TECH-323 extracted `backlog-record-schema.ts` pure `validateBacklogRecord`).
- `BlipTestFixtures.SampleEnvelopeLevels(buf, stride)` aliases oscillator phase when stride is not period-aligned; pick osc freq so `sampleRate/freq == stride` (e.g. 48 kHz / 3000 Hz = 16) so abs samples track envelope monotonically.
- `BlipPatch` SO has no `oscillatorCount` field — count derives from `oscillators.Length` in `BlipPatchFlat.FromSO`. Reflection-based `MakePatch` helpers must set the array, not a phantom int field.
- **Blip patch** SO canonical path: `Assets/Audio/Blip/Patches/BlipPatch_{Name}.asset` (not `Assets/Audio/BlipPatches/`). Filename prefix `BlipPatch_` mandatory for catalog wiring symmetry.
- `BlipFilterKind` MVP enum = `None=0`, `LowPass=1` only — HighPass semantics in §9 recipes (e.g. ToolRoadTick noise transient 4 kHz HP) encoded as `kind: None` + `cutoffHz` placeholder until post-MVP adds `HighPass=2`.
- `AgentBridgeCommandRunner.Mutations.cs` partial-class pattern (TECH-412) — bridge kind expansions live in a sibling partial class to isolate mutation dispatch from observation cases; keeps regression-gate diff reviewable. Reuse for future bridge surface additions.
- `JsonUtility.FromJson` + tagged-union via `value_kind` string + flat `string value` field works for polymorphic DTOs without pulling Newtonsoft.Json (TECH-412 `assign_serialized_field`). Keep DTOs `[Serializable]` + public fields; interpret at runtime in the switch.
- `AppDomain.CurrentDomain.GetAssemblies()` reflection resolves component type names uniformly across Territory + third-party assemblies (TECH-412). Return structured `type_ambiguous:<name>;candidates=<csv>` error on multi-hit instead of silent first-match.
- Next.js `serverExternalPackages` accepts npm package names only — for workspace-relative files outside `node_modules/` (e.g. `tools/progress-tracker/parse.mjs` imported via `../../`), the correct escape hatch is `outputFileTracingIncludes` mapping a route → file glob array. Default path: pure-ESM zero-dep files get inlined by server trace w/o any config.
- Sprite-gen palette extraction: when a class spec drives `apply_variant` material-family swaps (wall_brick_red ↔ wall_brick_grey, roof_tile_brown ↔ roof_tile_grey), bootstrap JSON must ship the grey-family slots up front — name 8 K-means clusters as `{wall_brick_red, wall_brick_grey, roof_tile_brown, roof_tile_grey, window_glass, concrete, trim, mortar}` instead of `{…, shadow, highlight, …}`, else `apply_variant` raises `PaletteKeyError` at render time.
- Smoke-style verification specs need Phase 0 deploy/state precondition check before HTTP probes — else blocked-on-deploy mislabels as smoke failure. Add `git status --porcelain web/` to web-platform kickoff preflight: untracked web files silently break Vercel parity (live `/dashboard` → 404 while local green).
- Next 16 `usePathname()` from `next/navigation` returns non-nullable `string` (signature changed vs Next 13/14 `string | null`). Check `node_modules/next/dist/client/components/navigation.d.ts` before writing null guards on App Router hooks.
- Next.js 16 App Router RSC forbids `ssr: false` inside `next/dynamic()` — build error `'ssr: false' is not allowed with next/dynamic in Server Components`. Wrap the dynamic import in a thin `'use client'` component (e.g. `FooClient.tsx` → `const Foo = dynamic(() => import('./Foo'), { ssr: false, loading })`) and import the client wrapper from the RSC. Canonical reference: `node_modules/next/dist/docs/01-app/02-guides/lazy-loading.md`.
- Node ≥22 `ts-node` / native TS strip mode drops `const enum` support — the inline pass cannot resolve identifiers across file boundaries. Use `as const` object pattern + derived type alias (`type Foo = typeof Foo[keyof typeof Foo]`) for portable enum-like constants in `tools/scripts/*.ts`.
- `__dirname` behaviour flips when ts-node reparses scripts as ESM — verify path resolution against the script file's actual depth before writing `resolve(SCRIPT_DIR, '..', '..', …)`. One `..` level from `tools/scripts/` lands in `tools/`; two levels escapes the repo root.
- `BlipPatch` SO `deterministic` flag gates jitter bypass, NOT rng seed — variant 0 with `deterministic:false` + jitter params all zero still yields a reproducible fixture because the seed derives from `(patchHash ^ variantIndex)` guard (`0x9E3779B9` fallback on zero).
- `Blip.Tests.EditMode.asmdef` sets `includePlatforms: ["Editor"]` — whole asmdef excluded from Standalone builds, so `AssetDatabase.LoadAssetAtPath<T>` calls inside these tests do NOT need `#if UNITY_EDITOR` guards. Applies to any EditMode-only asmdef w/ the same platform gate.
- Blip golden fixture tests re-use `BlipTestFixtures.RenderPatch(in flat, sampleRate, seconds, variantIndex)` — third arg is **seconds**, not sampleCount. Derive `seconds = fixture.sampleCount / fixture.sampleRate` w/ divisibility assert to fail fast when fixture schema drifts.
- `BlipTestFixtures` (Stage 1.4) is the single canonical helper class for Blip EditMode renders / hashes / zero-crossings — do not spawn parallel `BlipTestHelpers`; extend in place. Tolerance floor of 1e-6 on `sumAbsHash` drift matches `BlipDeterminismTests` epsilon.
- `GameNotificationManager` omits from EditMode fixtures — `Awake` NPEs on null `notificationPanel` serialized field (not assignable pre-Awake via `AddComponent`). For tests asserting insufficient-funds / error-post behavior, drop the manager from the fixture + assert return value + state unchanged instead of queue-count delta. Pattern applied in `Assets/Tests/EditMode/Economy/TreasuryFloorClampServiceTests.cs`.
- Feedback-comb kernel (`y[n] = x[n] + g·y[n−D]`) MUST write wet `y` to delay buffer, never dry `x` — dry write degrades kernel to single-tap delay (second echo = 0). Distinguishes feedback comb from chorus/flanger (wet-from-dry taps by design) and Schroeder allpass (writes `x + g·v`). Impulse-response invariant `out[2D]/out[D] ≈ g` is the audit check.
- Dashboard per-plan completion ratio counts from unfiltered `plan.allTasks` (not `plan.steps[*].stages[*].tasks`) — `filterPlans` prunes nested tasks only, so status / phase chips must not skew plan-level %. `DONE_STATUSES: ReadonlySet<TaskStatus>` covers both `'Done (archived)'` and short-form `'Done'`; single-string compare under-counts unarchived plans.
- Dashboard consumer-side StatBar wiring passes raw `value` + `max` counts — StatBar owns `(v/m)*100` + `[0,100]` clamp + `max ≤ 0 → 0` guard (TECH-232 contract). Do not pre-divide on the caller; reconcile any backlog snippet that shows `value={x / y * 100}` to `value={x} max={y}`.
- `@neondatabase/serverless` reached v1 stable (resolved `1.0.2` on 2026-04-16) — spec drafts citing `^0.9.x` are stale; always verify resolved version after `npm install` and record the actual pin in the Decision Log immediately.
- Proxy-based lazy export pattern (`new Proxy({} as NeonQueryFunction, { get, apply })` wrapping a `getSql()` getter) keeps tagged-template call-site ergonomics identical to eager `export const sql = neon(url)` while deferring the `process.env.DATABASE_URL` read until first invocation — confirmed safe through `next build` with no env set.
- Next.js 16 deprecated `web/middleware.ts` + `export function middleware` in favor of `web/proxy.ts` + `export function proxy` — dev server fails w/ `Cannot find the middleware module` until renamed. Any spec drafted against Next 13/14 middleware docs must migrate file + export name together; matcher config unchanged. Canonical reference: `node_modules/next/dist/build/webpack-config.ts` (middleware → proxy rename).
- Playwright exits 1 on "no tests found" by default — scaffold scripts (`test:e2e`, `test:e2e:ci`) must pass `--pass-with-no-tests` so an empty `tests/` dir does not break CI before the first spec lands.
- Plan-author must verify spec file paths against actual codebase — audit-sourced specs often cite stale filenames (e.g. `Cell.cs` when the type now lives in `CityCell.cs`). Grep the target symbol / class before writing `§Plan Author` sections; implementer agents will tick a checkbox without applying the edit if the cited path does not exist. Add `grep -rn "class {TargetType}" Assets/Scripts/` as a plan-author preflight when spec originates from a dated audit doc. (Rule originally written for retired `/kickoff`; same invariant applies to `plan-author` Stage 1×N.)
- Playwright SPA nav: prefer `page.waitForURL(/\/pattern\/.+/)` over `page.waitForResponse(...)` for click-then-navigate assertions — `waitForResponse` resolves on prefetch/asset responses before the SPA URL actually changes, leaving `page.url()` pointing at origin path. Assert destination URL, not network event.
- `npx playwright install chromium` is a one-time per-machine setup step separate from `npm install` — new Playwright harness (TECH-276 style) fails first `test:e2e` run with "Executable doesn't exist" until browsers are downloaded. Document in harness README + add preflight check if CI ever adopts.
- Meta-contract tests can surface missing prod SEO entries (robots `Disallow`, sitemap URLs, RSS `Content-Type`) — expect minimal prod fixes to land alongside "test-only" meta specs. Count them in Issues Found §9, not as scope creep.
- Manager-init race fix pattern: gate consumer's tick block (not whole `Update`) on producer's `IsInitialized` flag — keeps UI/input responsive during load while blocking sim-state reads. `[SerializeField]` producer ref + `FindObjectOfType` fallback in `Awake` (invariant #3). Sim `Update()` loops in `GridManager`/`UIManager` already self-guard; new race fixes usually self-contained.
- Smoke tests that mutate shared state (counters, lock files, sequence generators) must expose env-var path overrides on the mutator from day one — canonical pattern for `tools/scripts/reserve-id.sh` is `IA_COUNTER_FILE` + `IA_COUNTER_LOCK`; test scripts then `mktemp -d` + export overrides. Prevents production counter advance on first smoke run (real bug — reserve-id smoke advanced TECH 289→297 before isolation landed).
- Registry-count tests with strict equality (`expect(total).toBe(34)`) are a snapshot smell — untracked new files in the scanned dir break CI before first commit. Use lower-bound assertions (`expect(total).toBeGreaterThanOrEqual(34)`) for file counts that legitimately grow across unrelated sessions; covers `build-registry.test.ts` rule/spec totals.
- Add `npm run generate:ia-indexes` to the end of any skill / session that edits `ia/specs/glossary.md` or adds new spec sections — `line_start` caches shift with row inserts; `--check` mode fails CI otherwise. Easiest: run once after the last glossary edit in a given phase.
- `flock` guard on read-only validators (`validate:dead-project-specs`, `validate:backlog-yaml`) adds maintenance cost for zero safety gain — reads cannot corrupt yaml/state; worst race outcome = redundant work, not lost data. Skip the guard when a validator is read-only, note rationale inline.
- Lockfile-per-domain rule (TECH-330 C5, 2026-04-17) — `flock` targets must be 1:1 with their concurrency domain. Id-counter + closeout are distinct domains → separate `.id-counter.lock` + `.closeout.lock`; reusing one file wedges the other when the holder crashes. Apply same rule to `materialize-backlog.sh` (`.materialize-backlog.lock`). Mutation scripts only — read-only validators skip per rule above.
- `(f)` tracker cell predicate (TECH-330 C3, 2026-04-17) — canonical signal for "≥1 task filed" in `{umbrella-slug}-rollout-tracker.md` column (f) = paired **backlog record** (`ia/backlog/{id}.yaml`) AND project spec (`ia/projects/{id}*.md`) both present → `✓`; backlog record without matching spec → `◐`; zero records → `—`. Both `release-rollout-enumerate` + `release-rollout-track` must share this predicate verbatim; divergent grep-based heuristics auto-regress `✓ → —` on re-scan.
- `--reserved-id {ID}` flag contract (TECH-330 C2, 2026-04-17) — `stage-file` is the single writer to `reserve-id.sh` per task chain; batch-reserves N ids + forwards `--reserved-id {ID}` to each `project-new` delegation. `project-new` skips `reserve-id.sh` when the arg is set + consumes the id verbatim. Preserves invariant #13 (one writer per chain); avoids dual-reserve race under concurrent `/stage-file`.
- `backlog-yaml-loader` parse-error preemption (TECH-410, 2026-04-18) — `loadAllYamlIssues` validates `task_key` regex during load and skips the record on failure (`parseErrorCount++`); the validator-side TASK_KEY_RE check at `tools/mcp-ia-server/src/parser/parent-plan-validator.ts:194` is unreachable via the loader path. Fixture tests document actual behavior (empty records → `errors=[], warnings=[], exit_code=0`) rather than spec §4.3 expected `errors=[1 match "fails /^T..."]`. No code fix needed — parse-error count is the correct surface.
- Backlog-archive yaml `status:` enum = `closed` only (TECH-458, 2026-04-19) — schema rejects `completed` / `done` / `archived`; prior closeouts shipped invalid `status: completed` values. Validate archive yamls w/ `validate:backlog-yaml` after closeout; fix by replacing with `status: closed`.
- Zod v3 enum rejection `code` = `invalid_value` (TECH-458, 2026-04-19) — NOT `invalid_enum_value` (that's Zod v4). Handler unit tests asserting enum rejection should accept either code or match on message/path instead of hard-coded `code` literal to stay portable across the v3→v4 bump.
- `code-review-plan` skill yaml+row contract (TECH-330 §6, 2026-04-17; updated 2026-04-19 rev 3) — any skill authoring `ia/projects/{id}.md` MUST also file `ia/backlog/{id}.yaml` + run `tools/scripts/materialize-backlog.sh`; otherwise `/ship-stage` per-task lookup breaks (empty `backlog_issue` MCP lookup) + Stage-scoped `/closeout` has no row to archive. TECH-330 itself hit this gap — spec shipped with reserved id but no yaml; closeout had to back-fill the row. Make the dual-write explicit in the skill body.
