# Archived specs and plans

This folder holds **completed** implementation summaries, scene-setup notes, and **historical** agent prompts that are **not** part of active development documentation.

## For AI agents and contributors

- **Do not** read these files by default when working on new issues — they add token cost and can look like open work.
- Use **active** docs: `ARCHITECTURE.md`, `BACKLOG.md`, `.cursor/specs/*.md` outside this folder, and `docs/` (e.g. active plans such as `docs/plan-zoning-road-candidates-grass-forest-slopes.md`).
- Open this folder only when you need **historical context** (e.g. how BUG-29 cut-through was implemented, FEAT-17 minimap setup, load-game sorting prompts).
